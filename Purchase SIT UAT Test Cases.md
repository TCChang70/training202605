## 1. 測試目標

1. 驗證高優先需求功能完整可用。
2. 驗證採購主流程端到端正確性。
3. 驗證邊界與例外處理符合業務規則。
4. 驗證權限與安全控制符合角色設計。
5. 驗證上線門檻：Critical 缺陷 0 件。

---

## 2. 測試範圍

### 2.1 In Scope

1. 產品查詢
2. 庫存警示
3. 採購申請建立與送審
4. 採購申請核准與退回
5. 採購訂單建立、狀態變更與取消
6. 收貨驗收與庫存更新
7. 報表摘要查詢
8. RBAC 權限與 API 驗證

### 2.2 Out of Scope

1. 供應商外部入口
2. 財務分錄與發票整合
3. 手機 App 原生功能

## 4. 測試進出條件

### 4.1 SIT 進入條件

1. API 開發完成度達 100%（高優先需求）。
2. 單元測試通過率 >= 90%。
3. 測試環境與測試資料已部署完成。

### 4.2 SIT 結束條件

1. SIT 測試案例執行率 >= 95%。
2. Critical 缺陷 0 件。
3. Major 缺陷已修復或具備可接受 workaround。

### 4.3 UAT 進入條件

1. SIT 結束條件達成。
2. 業務測試劇本已核可。
3. UAT 角色帳號與權限配置完成。

### 4.4 UAT 結束條件

1. 高優先需求驗收通過率 100%。
2. 業務單位簽核完成。
3. 上線門檻達成（Critical 0 件，Major <= 3 件）。

## 7. 缺陷分級與處理 SLA

| 等級 | 定義 | 處理時限 |
|------|------|----------|
| Critical | 核心流程無法執行或資料錯誤 | 24 小時內修復 |
| Major | 主要功能受影響但可繞過 | 3 個工作日 |
| Minor | 次要功能或顯示問題 | 7 個工作日 |
| Trivial | 文案或樣式微調 | 版本合併修正 |
